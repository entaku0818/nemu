# iOS アプリリリースチェックリスト

## リリース前（開発完了時）

### コード品質
- [ ] すべてのテストが通過している
- [ ] コードレビューが完了している
- [ ] 未解決のバグがない（またはリリースブロッカーがない）
- [ ] デバッグコードやconsole.logが削除されている
- [ ] コミットされていない変更がない（`git status` でクリーン）

### ビルド設定
- [ ] Release ビルド設定でビルドが成功する
- [ ] アプリアイコンが正しく設定されている
- [ ] Launch Screenが正しく表示される
- [ ] バージョン番号が適切に更新されている
- [ ] Bundle Identifierが正しい

### 証明書とプロビジョニング
- [ ] App Store Distribution 証明書が有効
- [ ] App Store プロビジョニングプロファイルが有効
- [ ] fastlane match が正しく設定されている（使用している場合）

### App Store Connect
- [ ] アプリの基本情報が最新（名前、説明、カテゴリなど）
- [ ] スクリーンショットが最新（全デバイスサイズ）
- [ ] プレビュー動画が準備できている（オプション）
- [ ] キーワードが最適化されている
- [ ] サポートURLとプライバシーポリシーURLが正しい
- [ ] 年齢レーティングが適切

## TestFlightリリース時

### ビルド準備
- [ ] ビルド番号を確認（前回より大きい値）
- [ ] リリースノートを準備
- [ ] テスト観点リストを作成

### アップロード
- [ ] `fastlane beta` でアップロード成功
- [ ] App Store Connectでビルドのステータスが「処理中」になっている
- [ ] 10-15分後に「テスト可能」になったことを確認

### テスター通知
- [ ] 内部テスターに通知（自動または手動）
- [ ] 外部テスターに通知（承認後）
- [ ] テストフィードバックチャンネルを開設（Slack、メールなど）

### テスト実施
- [ ] インストールが成功する
- [ ] 主要な機能が動作する
- [ ] クラッシュが発生しない
- [ ] パフォーマンスに問題がない
- [ ] 様々なデバイスでテスト（iPhone、iPad）
- [ ] 異なるiOSバージョンでテスト

## App Store本番リリース時

### 最終確認
- [ ] TestFlightでの十分なテスト期間（推奨: 3-7日）
- [ ] 重大なバグ報告がない
- [ ] プロダクトオーナー/ステークホルダーの承認取得
- [ ] リリースノートの最終レビュー

### メタデータ
- [ ] 「このバージョンの新機能」を更新
- [ ] すべての言語でローカライズ（多言語対応の場合）
- [ ] スクリーンショットが最新機能を反映
- [ ] プロモーション用テキストを更新（オプション）

### App Store Connect設定
- [ ] 価格設定の確認
- [ ] 配信可能地域の確認
- [ ] 段階的リリースの設定（推奨）
- [ ] アプリ内購入の設定確認（該当する場合）

### アップロード
- [ ] `fastlane release` でアップロード成功
- [ ] App Store Connectで「レビュー待ち」ステータス確認
- [ ] 必要に応じてレビュー用メモを追加（特殊な操作が必要な場合）
- [ ] 連絡先情報が正しい

### Git管理
- [ ] バージョン更新をコミット
- [ ] タグを作成（例: `v2.1.0`）
- [ ] リモートにプッシュ
- [ ] GitHub Releaseを作成（オプション）

## レビュー申請後

### モニタリング
- [ ] App Store Connectでステータスを定期的に確認
- [ ] Appleからの問い合わせに迅速に対応
- [ ] レビュー却下時の対応計画準備

### 承認後
- [ ] リリース日時を設定（自動リリースまたは手動）
- [ ] 段階的リリースの進行状況を監視
- [ ] クラッシュレポートを確認（Crashlytics、Sentryなど）
- [ ] ユーザーレビューを監視
- [ ] アナリティクスで採用率を確認

## リリース後（公開後）

### 即時対応
- [ ] App Storeで正しく表示されることを確認
- [ ] 実際にダウンロード・インストールしてテスト
- [ ] クラッシュレポートを24時間監視
- [ ] ユーザーからの問い合わせに対応

### コミュニケーション
- [ ] プレスリリース（大型アップデートの場合）
- [ ] ブログ記事の公開
- [ ] SNSでアナウンス（Twitter、Facebook、Instagramなど）
- [ ] メールニュースレター送信（ユーザーベースがある場合）
- [ ] 社内に報告

### 監視期間（1-2週間）
- [ ] デイリーアクティブユーザー（DAU）の変化
- [ ] クラッシュ率の推移
- [ ] ユーザーレビューとレーティング
- [ ] パフォーマンス指標（起動時間、APIレスポンスなど）
- [ ] App Store検索ランキング

### 次回への振り返り
- [ ] うまくいった点を記録
- [ ] 改善点を洗い出し
- [ ] リリースプロセスの改善提案
- [ ] ドキュメントの更新

## 緊急時の対応

### クリティカルなバグが見つかった場合
1. [ ] 影響範囲の特定
2. [ ] 段階的リリースの停止（可能な場合）
3. [ ] ホットフィックスブランチの作成
4. [ ] 修正版のビルドとテスト
5. [ ] 緊急リリースの申請（Expedited Reviewのリクエスト）

### Apple審査で却下された場合
1. [ ] 却下理由を詳細に確認
2. [ ] 必要な修正を実施
3. [ ] Resolution Centerで対応内容を説明
4. [ ] 再申請

## 便利なコマンド集

```bash
# ビルド番号の確認
fastlane run get_build_number

# バージョン番号の確認
fastlane run get_version_number

# 最新のTestFlightビルド番号を取得
fastlane run latest_testflight_build_number

# App Store Connectのステータス確認
fastlane run get_version_number_from_git_branch

# スクリーンショット生成
fastlane snapshot

# メタデータのダウンロード
fastlane deliver --download_metadata

# メタデータのアップロード（バイナリなし）
fastlane deliver --skip_binary_upload
```

## 参考タイムライン

| 段階 | 所要時間 |
|------|---------|
| TestFlightアップロード | 10-15分 |
| TestFlightでのテスト | 3-7日（推奨） |
| App Storeレビュー申請 | 数分 |
| レビュー待ち | 24-48時間（平均） |
| レビュー中 | 数時間〜1日 |
| 承認後リリース | 即時〜指定日時 |
| 段階的リリース | 7日間（デフォルト） |

**合計**: 最短で4-5日、通常は1-2週間を見込む
