# Nemu（ねむ）— デザインドキュメント

> 「確実に起きられる、気持ちいい目覚め」
> 寝る〜起きるまで、まるごとサポートする睡眠アプリ

---

## 1. 概要

| 項目 | 内容 |
|---|---|
| アプリ名 | Nemu（ねむ） |
| コンセプト | 寝る〜起きるまで全部サポート |
| ターゲット | 忙しいビジネスパーソン |
| 世界観 | ナチュラル・有機的（自然・森・月） |
| トーン | やさしく提案。押しつけない、でも気にかけてる |
| OS | iOS（iPhone単体） |

### アプリ名の由来

- ネムノキ = 夜になると葉を閉じて眠る木。コンセプトと完全一致
- 「眠る」の意味もそのまま
- 短くて覚えやすい、App Storeで映える

---

## 2. コアバリュー

1. **確実に起きられる** — AlarmKitでサイレントモードでも絶対鳴る
2. **自然に寝られる** — 入眠サポートが手厚い
3. **やさしくダメ出し** — データは見せるが、判断はユーザーに委ねる
4. **シンプルで迷わない** — アプリを開く時間すら惜しいビジネスパーソン向け

---

## 3. アプリの全体フロー

```
夜（就寝前）
├ 就寝リマインダー通知（UserNotifications）
├ 今日のストレス・感謝を3行メモ
├ 呼吸法ガイド（478呼吸法）
├ 自然音 + フェードアウトタイマー（AVFoundation）
└ 画面が自動で暗くなる（UIScreen.brightness）

就寝中
├ CMMotionActivityManager → 動き検知（眠りの深さ）
├ UIScreen.brightness → 朝の光を間接検知
└ CoreLocation → 日の出時刻を監視

起床（3条件トリガー）
├ ① 日の出前後30分以内（CoreLocation）
├ ② 動きが増えてきた（CoreMotion）
├ ③ 画面輝度が上がってきた（UIScreen.brightness）
└ AlarmKit → サイレントでも確実に鳴らす

起床後
├ Dynamic Island → 睡眠スコア表示
├ WeatherKit → 今日の天気・気温を表示
└ HealthKit → 睡眠データを保存・記録
```

## 4. 技術スタック

| フレームワーク | 用途 | 備考 |
|---|---|---|
| **AlarmKit** | サイレントでも確実に鳴らすアラーム | iOS 26〜 |
| **CoreMotion** | 動き検知・睡眠ステージ判定 | CMMotionActivityManager |
| **SensorKit** | 本物のLux値で照度検知・睡眠セッション | Apple申請が必要 |
| **HealthKit** | 睡眠データの読み書き・保存 | |
| **WeatherKit** | 外気温・天気・日の出時刻 | |
| **CoreLocation** | 位置情報から日の出時刻を取得 | |
| **AVFoundation** | 自然音再生・マイクで騒音検知 | |
| **UserNotifications** | 就寝リマインダーなど補助通知 | |

### AlarmKitでできること

- サイレント・フォーカスモードでも確実に鳴らせる
- 繰り返し設定（例：毎週月〜金）
- Dynamic Island・ロック画面・Apple Watchに対応
- カウントダウン表示（一時停止・再開・キャンセル）
- ボタンのテキスト・色・アイコンを自由にカスタマイズ
- カスタムサウンド（ローカルファイルのみ）
- App Intentsでボタンタップ時に独自コードを実行

### SensorKit申請戦略

| フェーズ | 内容 |
|---|---|
| フェーズ1（リリース時） | UIScreen.brightnessで間接的に照度検知 |
| フェーズ2（申請中） | AppleへSensorKit申請を提出 |
| フェーズ3（承認後） | 高精度Lux値＋sleepSessionsで精度大幅UP |

> SensorKitの`sleepSessions`センサーが使えれば、CoreMotionで自前実装しようとしていた睡眠ステージ判定がそのまま代替可能。

## 5. 機能一覧

### 入眠サポート（就寝前）

- **就寝リマインダー** — 設定時間になるとやさしく通知
- **3行メモ** — 今日のストレス・感謝を書いて頭を空っぽに
- **呼吸法ガイド（478呼吸法）** — SwiftUIアニメーションで視覚的に誘導
- **自然音タイマー** — ホワイトノイズ・波・森など。フェードアウトで消える
- **自動減光** — 就寝ボタンで画面が月明かりのように暗くなる

### 就寝中

- CMMotionActivityManagerで「静止 → 動き始め」を検知
- 設定した起床時間の前後30分以内で最適タイミングを判断
- UIScreen.brightnessで朝の光の変化を監視
- CoreLocationで今日の日の出時刻を取得・監視

### アラーム（AlarmKit）

- **3条件トリガー** — 日の出 × 動き × 明るさが重なったとき起動
- **サイレントでも確実に鳴る**
- **「起きた！」ボタン** — App Intentで今日の天気・予定を表示
- **「あと5分…」ボタン** — カスタムスヌーズ
- **カスタムサウンド** — 自然音から徐々に音楽へ
- **繰り返し設定** — 毎週月〜金など

### 起床後レポート

- 睡眠スコア・動いた回数・目覚めた時間
- Dynamic Island・ロック画面・ウィジェットに表示
- HealthKitに睡眠データを保存

### 睡眠環境のやさしいダメ出し（プレミアム）

朝のレポートにさりげなく添える。毎日しつこく出さない（週1程度）。

```
今夜のヒント

部屋の明かりをもう少し落とすと入眠が早くなるかも

スマホは寝る20分前に置いてみると変わるかもです

今日は晴れ 18℃ / 傘は不要です
```

検知できる環境データ：

| 項目 | 手段 |
|---|---|
| 明るさ | UIScreen.brightness（間接）/ SensorKit（高精度・申請後） |
| 騒音 | AVFoundationのマイクで音量レベルをサンプリング |
| 外気温 | WeatherKit |
| スマホ使用時間 | Screen Time API |

## 7. 競合との差別化

| アプリ | 強み | Nemuとの違い |
|---|---|---|
| Calm | ブランド・瞑想コンテンツ | 睡眠分析が弱い |
| AutoSleep | Apple Watch連携・精度 | iPhone単体で動かない |
| Sleep Cycle | 睡眠サイクル分析 | AlarmKitに非対応（サイレントで鳴らない） |

**Nemuの独自ポイント：**
- 日の出 × 動き × 明るさの3条件トリガー → 既存アプリにない
- AlarmKitでサイレントでも絶対鳴る
- SensorKit対応後は高精度Lux値で睡眠環境を分析
- 「寝かせる」側の体験が手厚い

---

## 8. 既知の制限・課題

| 課題 | 対応方針 |
|---|---|
| iPhone単体では睡眠精度に限界がある | 「完璧な分析より、確実に起きられる」と正直に伝える |
| SensorKitはApple申請が必要 | フェーズ1はUIScreen.brightnessで代替。申請を並行して進める |
| AlarmKitでスワイプ消去の検知ができない | 起床確認機能は別途工夫が必要 |
| バックグラウンドでのセンサー常時稼働 | Background ModesとBGタスクで対応 |
| カスタムサウンドはローカルファイルのみ | ストリーミング音楽は非対応。自然音を同梱する |
